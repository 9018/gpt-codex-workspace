# P2 — 拓展性、插件化与状态演进

## 目标

为长期团队协作、多 workspace、多 agent、多插件做结构准备。

## 必做项

### 1. Tool registry metadata 化

当前 `createTool(description, inputSchema, handler)` 太薄。

建议扩展：

```js
createTool({
  name,
  description,
  inputSchema,
  handler,
  audience: ["chatgpt", "codex", "operator"],
  modes: ["standard", "full"],
  outputCard: "runtimeStatusCard",
  examples: [],
  tags: ["goal", "workspace", "diagnostics"]
})
```

### 2. Schema builder 升级

当前 `schema({ path: "string" })` 只能表达简单类型。

建议：

- 支持 enum
- 支持 default
- 支持 descriptions
- 支持 array items
- 支持 oneOf
- 支持 examples

可选实现：

- 自研轻量 JSON Schema builder
- 或迁移到 zod-like builder，再输出 JSON Schema

### 3. 状态存储演进

当前 JSON + indexes 对小规模可行。进入多 agent 后建议：

阶段一：

- 保持 JSON，但增加 event log：
  - `.gptwork/events/*.jsonl`
  - append-only task/goal/agent events

阶段二：

- SQLite：
  - tasks
  - goals
  - conversations
  - memories
  - agent_runs
  - artifacts
  - events

### 4. Plugin/hook 机制

新增 hook points：

```text
onGoalCreated
onTaskAssigned
onAgentRunStarted
onAgentRunCompleted
onTaskCompleted
onGithubIssueSynced
onNotification
onContextBuild
```

用途：

- notification adapters
- GitHub/GitLab/Linear adapters
- agent adapters
- context loaders
- card renderers

### 5. Apps SDK widget

实现真正的 ChatGPT visual card：

- `ui://widget/gptwork-card-v1.html`
- tool descriptor `_meta["openai/outputTemplate"]`
- compact UI for:
  - create_encoded_goal
  - runtime_status
  - worker_status
  - get_task
  - preview_codex_context
  - github_status
  - show_changes

## P2 验收标准

- 工具注册支持 metadata filtering。
- 至少 3 个工具使用 richer schema。
- event log 能记录 task/goal 生命周期。
- 至少一个 hook 被 notification 或 GitHub sync 使用。
- 至少一个 Apps SDK widget card 可渲染。
