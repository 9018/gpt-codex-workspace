# P0 — 使用体验与入口产品化

## 目标

让普通用户和 agent 不需要阅读 README 大段说明，也能完成 setup、启动、诊断、创建 goal、查看 worker 状态。

## 必做项

### 1. 新增 `gptwork` CLI

建议新增：

```bash
gptwork setup
gptwork start
gptwork status
gptwork doctor
gptwork settings show
gptwork settings set
gptwork logs
gptwork goal create --assign
gptwork codex run --limit 5
gptwork github sync
```

建议文件：

- `backend/bin/gptwork.mjs` 或 `bin/gptwork.mjs`
- `backend/src/cli/*` 或 `backend/src/gptwork-cli-*.mjs`
- `backend/package.json` 增加 `bin`
- README 改成 CLI-first quick start

### 2. 增加 tool mode

新增配置：

```bash
GPTWORK_TOOL_MODE=minimal|standard|full|operator|codex
```

默认建议：`standard`

模式建议：

| 模式 | 暴露工具 |
|---|---|
| minimal | health/status、create_encoded_goal、get_task、list_tasks |
| standard | goal/task、preview、workspace read、show result、github sync |
| operator | restart、repo lock、runtime env、diagnostics |
| codex | get_goal_context、append_goal_message、workspace ops、run task |
| full | 所有兼容/调试工具 |

建议改动：

- `backend/src/server-tools.mjs`
- `backend/src/runtime-config.mjs`
- `backend/src/mcp-tooling.mjs`
- 每个 `tool-groups/*.mjs` 增加 group metadata 或集中注册过滤

### 3. 新增 `open_project_context`

目的：ChatGPT 第一步就能获得项目状态，而不是让用户理解内部 task/goal 机制。

返回内容：

- repo 摘要
- branch/head/dirty state
- project.md/project.env 状态
- AGENTS.md / SKILL.md discovery
- 最近 tasks/goals 摘要
- worker status
- recommended next tools
- bounded file tree
- discovered scripts/test commands

建议新增：

- `backend/src/tool-groups/project-context-tools-group.mjs`
- `backend/src/project-context-service.mjs`

### 4. 低噪声输出

- 默认 compact。
- 原始 logs、diffs、stdout/stderr 不直接刷屏。
- 返回 artifact paths / structuredContent。
- `get_task`、`shell_exec`、`runtime_status`、`worker_status`、`create_encoded_goal` 使用一致卡片摘要。

### 5. README 改为 CLI-first

README 首屏建议变成：

```bash
npm install -g gptwork
gptwork setup
gptwork start
gptwork doctor
```

底层 systemd/env/反代说明移到 operator docs。

## P0 验收标准

- `gptwork doctor` 能在本机输出健康检查摘要。
- `gptwork status` 能显示服务、worker、repo、queue、GitHub sync 状态。
- MCP `tools/list` 在默认 mode 下不再暴露所有工具。
- ChatGPT 能用 `open_project_context` 完成第一步项目理解。
- 创建 goal 后返回短摘要，不刷出完整 payload/context。
