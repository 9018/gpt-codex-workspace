# P1 — 协作、多 agent 与 handoff

## 目标

把当前 prompt-level 的 subagent 约定升级为可追踪、可复用、可审计的协作执行模型。

## 必做项

### 1. 新增 `agent_runs` 数据模型

建议结构：

```json
{
  "id": "agent_run_xxx",
  "goal_id": "goal_xxx",
  "task_id": "task_xxx",
  "role": "planner|architect|implementer|tester|reviewer|finalizer",
  "agent": "codex|opencode|pi|custom",
  "status": "queued|running|completed|failed|waiting_for_review",
  "input_artifacts": [],
  "output_artifacts": [],
  "summary": "",
  "created_at": "",
  "updated_at": ""
}
```

### 2. 新增 agent tools

建议工具：

```text
create_agent_run
list_agent_runs
get_agent_run
append_agent_event
complete_agent_run
run_agent_pipeline
handoff_to_agent
read_handoff
show_changes
```

### 3. 借鉴 CodexPro handoff/watch-handoff

新增文件约定：

```text
.gptwork/handoff/current-plan.md
.gptwork/handoff/agent-status.md
.gptwork/handoff/implementation-diff.patch
.gptwork/handoff/execution-log.jsonl
```

CLI：

```bash
gptwork watch-handoff --agent codex
gptwork watch-handoff --agent opencode
gptwork watch-handoff --agent custom --command "node ./agent.js --task-file {{plan_file}}"
```

### 4. Review gate

在多 agent pipeline 中引入 review gate：

- planner/architect 只出计划。
- implementer 修改代码。
- tester 跑验证。
- reviewer 总结 diff 和风险。
- finalizer 写 `result.json`、goal transcript 和 task result。

### 5. GitHub Issues 协作增强

- Issue comment 映射为 event。
- 每个 agent run 可写一条 progress comment。
- final result comment 使用 compact summary。
- raw diff/log 作为 artifact path。

## P1 验收标准

- 一个 goal 可拆成多个 agent run。
- agent run 状态能被查询。
- `handoff_to_agent` 能写计划文件。
- `gptwork watch-handoff` 能检测新 plan 并启动本地 agent 或 dry-run。
- `show_changes` 能输出统一 review 摘要。
